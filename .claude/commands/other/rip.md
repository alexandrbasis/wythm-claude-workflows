# Review Implementation Plan (RIP)

Пошаговый разбор имплементационного плана. Фокус: **бизнес-ценность** через техническое понимание.

## Протокол

### Фаза 1: Анализ

Перед ответом:
1. Прочитать план полностью
2. Изучить связанный код (Grep, Read) — проверить, не предполагать
3. Сравнить с PRD/бизнес-требованиями — выявить несоответствия
4. Определить бизнес-ценность каждого шага

<validation>
Если найдены несоответствия (лишние фичи, пропущенные требования, конфликты логики) — зафиксировать для обсуждения. Критика = факты + вопрос.
</validation>

**Формат обзора:**

```
## Обзор

**Бизнес-цель:** [какую проблему решает]
**Скоуп:** [компоненты] | [файлы] | [сложность]
**План:** [N шагов с названиями]

**⚠️ Вопросы** (если есть):
- [Проблема] — противоречит [источник]

Готов к первому шагу?
```

### Фаза 2: Разбор

<discipline>
Переходить к следующему шагу только после подтверждения ("готов", "дальше"). Для каждого шага проверять соответствие PRD — при несоответствии запросить подтверждение.
</discipline>

**Формат шага:**

```
### Шаг [N]: [Название]

**Зачем:** [бизнес-причина]
**Что:** [техническая суть]
**Концепции:** [термин]: [аналогия]
**Влияние:** [код] → [изменение для юзера]
**Код:** [если релевантно]

**⚠️ Внимание** (если есть):
- [Проблема с PRD]
- **Вопрос:** [уточнение]

---
Вопросы? Дальше?
```

### Фаза 3: Итоги

```
## Итоги

**Разобрали:** [резюме]
**Ценность:** [зачем пользователю]
**Концепции:** [список]

**Самопроверка:**
1. [Бизнес-вопрос]
2. [Технический вопрос]
```

## Правила

| Принцип | Действие |
|---------|----------|
| Бизнес-first | Начинать с "зачем пользователю" |
| Простота | Термин = аналогия |
| Конкретика | Реальные файлы/эндпоинты |
| Прогресс | Каждые 3-4 шага: "X из N" |
| Критичность | Указывать несоответствия PRD **только** когда есть |
| Конструктивность | Факты + вопрос, без обесценивания |

## Пример

**Input:** План трекинга прогресса

**Обзор:**

> **Бизнес-цель:** Юзеры видят прогресс → мотивация
> **Скоуп:** backend + DB | `src/modules/progress/` | средняя
> **План:** 4 шага (модель, API, логика, интеграция)
>
> **⚠️ Вопросы:**
> - План не упоминает визуализацию — PRD требует графики. За скоупом?
> - PRD: экспорт в PDF — в плане нет. Включить?

**Шаг 1:**

> **Зачем:** Хранить историю изучения слов
> **Что:** Таблица `WordProgress` связывает юзера+слово, хранит метрики
> **Концепции:** Prisma schema = чертёж БД как код
> **Влияние:** Экран "Прогресс" → статистика по словам
>
> **⚠️ Внимание:**
> - PRD: отслеживать интервалы повторений (spaced repetition). Схема: только `lastSeen`, нет истории.
> - **Вопрос:** План — только текущее состояние или полная история?
