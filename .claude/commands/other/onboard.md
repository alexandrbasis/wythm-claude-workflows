## ОСНОВНАЯ ЗАДАЧА
Провести пошаговый онбординг junior-разработчика в проект с нуля до успешного локального запуска.

## КОНТЕКСТ
- Уровень: junior разработчик, минимальный опыт с похожими проектами
- Текущее состояние: нет понимания архитектуры, технологий, процессов
- Цель: самостоятельный запуск проекта локально и понимание базовой структуры

## ФОРМАТ РАБОТЫ

### Шаг 1: Создать План Онбординга (MD файл)
Структура плана должна включать:

**1. Environment Setup**
- [ ] Установка необходимых инструментов (список с версиями)
- [ ] Настройка окружения
- [ ] Проверка работоспособности (команды для самопроверки)

**2. Project Structure**
- [ ] Обзор директорий (что где лежит и зачем)
- [ ] Основные файлы конфигурации
- [ ] Точки входа в приложение

**3. Database Setup**
- [ ] Установка БД
- [ ] Создание схемы
- [ ] Загрузка тестовых данных
- [ ] Проверка подключения (SQL-команда для теста)

**4. First Run**
- [ ] Запуск приложения
- [ ] Проверка работы (URL/эндпоинт для проверки)
- [ ] Типичные проблемы и их решения

### Правила Подачи Информации

**Размер порции:**
- Максимум 10 команд или концептов за раз
- Каждый блок должен заканчиваться проверкой понимания
- Переход к следующему шагу ТОЛЬКО после явного подтверждения

**Формат каждого шага:**
```
## Шаг X: [Название]

### Что делаем и зачем
[1-2 предложения о цели]

### Действия
1. [Команда/действие с объяснением]
2. [Команда/действие с объяснением]

### Проверка понимания
- [ ] Выполнил все команды без ошибок
- [ ] Понимаю, зачем это нужно
- [ ] Получил ожидаемый результат: [конкретный результат]

### Типичные проблемы
- Ошибка X → Решение Y
```

## КРИТЕРИИ УСПЕХА ШАГА
Шаг считается завершенным, когда junior:
1. ✅ Выполнил все команды
2. ✅ Получил ожидаемый результат (скриншот/вывод команды)
3. ✅ Может объяснить своими словами, что произошло

## ПРАВИЛА ОБЩЕНИЯ
- Жду явного "готов к следующему шагу" перед продолжением
- При любой ошибке — останавливаемся, разбираем, исправляем
- Избегаю технического жаргона без объяснения
- Каждый новый термин объясняю простым языком

## НАЧАЛО РАБОТЫ
Создай MD файл с планом онбординга для этого проекта. Структура должна соответствовать шаблону выше.
После создания плана жди моего подтверждения перед началом Шага 1.
